# Лабораторная работа N4: ленивые последовательности и потоки

Проект реализует ленивую последовательность `LazySequence<T>`, способную
работать как с конечными, так и с бесконечными наборами данных, а также
набор потоков для последовательного чтения и записи элементов.

В качестве основы используются контейнеры и интерфейсы из второй
лабораторной работы: `Sequence`, `MutableArraySequence`, `ArraySequence`,
`LinkedList` и связанные классы. В новых классах четвёртой лабораторной
владение создаваемыми объектами оформлено через умные указатели.

В проекте не используется `std::vector`.

## Возможности

### Ordinal

Класс `Ordinal` представляет длину последовательности:

- `Ordinal::Finite(n)` - конечная длина;
- `Ordinal::Omega()` - бесконечная длина;
- сравнение конечных значений и `Omega`;
- проверка наличия индекса через `ContainsIndex`;
- сложение длин с учётом бесконечности.

`Omega` нельзя использовать как конкретный индекс элемента и нельзя
преобразовать в обычное число через `Value()`.

### LazySequence

`LazySequence<T>` вычисляет элементы только при обращении к ним.
Полученные значения сохраняются во внутреннем кэше, поэтому повторный
доступ к уже вычисленному префиксу не запускает генерацию заново.

Создание последовательности поддерживает:

- пустую последовательность;
- массив или обычную `Sequence<T>`;
- генерацию элемента по индексу;
- рекуррентную генерацию по уже вычисленному префиксу;
- конечную длину или `Omega`.

Основные операции:

- `Get`, `GetFirst`, `GetLast`;
- `GetSubsequence`, в том числе бесконечный хвост;
- `Append`, `Prepend`, `InsertAt`;
- `RemoveAt`, `RemoveRange`;
- `Concat`;
- `Map`, `Where`;
- `Reduce`, `ReduceFirstN`;
- `Take`;
- `GetEnumerator`.

Операции, создающие новую ленивую последовательность, возвращают
`std::unique_ptr<LazySequence<T>>`. Внутреннее состояние и связи между
ленивыми операциями используют `std::shared_ptr`.

`Zip` не входит в API `LazySequence` текущей версии; одноимённый старый метод
остаётся только в базовом `Sequence` из второй лабораторной работы.

### Потоки

В файле `Streams.h` реализованы:

- `SequenceReadStream<T>` - чтение из обычной последовательности;
- `LazySequenceReadStream<T>` - чтение из ленивой последовательности;
- `StringReadStream<T>` - чтение элементов из строки;
- `FileReadStream<T>` - построчное чтение из файла;
- `SequenceWriteStream<T>` - запись в последовательность;
- `FileWriteStream<T>` - запись элементов в файл.

Потоки чтения поддерживают:

- `Open` и `Close`;
- `Read`;
- `Seek`;
- `GetPosition`;
- `IsEndOfStream`;
- `IsCanSeek` и `IsCanGoBack`.

Потоки последовательностей хранят источники и назначения через
`std::shared_ptr`. Для объектов, созданных на стеке, доступны конструкторы
от ссылки: поток использует объект, но не удаляет его.

## Консольный интерфейс

Программа запускает русскоязычный интерактивный UI для ручного
исследования реализованных классов.

Главное меню содержит:

1. Работа с конечной `LazySequence`.
2. Создание `LazySequence` через генератор.
3. Поток чтения обычной последовательности.
4. Поток чтения `LazySequence`.
5. Поток чтения строки.
6. Поток записи в последовательность.
7. Файловые потоки.
8. Встроенные автоматические проверки.
9. Нагрузочная проверка `LazySequence`.
10. Нагрузочная проверка файловых потоков.

В рабочем разделе `LazySequence` можно пошагово менять текущую
последовательность и проверять все основные операции: добавление,
удаление, вставку, объединение, фильтрацию, отображение, свёртку,
подпоследовательности, материализацию и перечислитель.

Доступные генераторы:

- числа Фибоначчи;
- арифметическая прогрессия;
- геометрическая прогрессия;
- квадраты натуральных чисел;
- кубы натуральных чисел;
- факториалы;
- формула `a*n + b`;
- натуральные числа.

## Вариант 3.3: прогноз / коррекция

Основное задание реализовано как мониторинг числовых событий. Событие
содержит номер, время, тип, значение и имя источника:

- `CPU_LOAD`;
- `MEMORY_LOAD`;
- `TEMPERATURE`;
- `REQUEST_ERRORS`.

Новый прикладной слой находится в `classes/headers/ForecastCorrection.h`.
Он включает:

- `Event`, `Prediction`, `Correction`, `Reaction`, `ProcessingResult`;
- `HistoryBuffer` - ограниченную историю фактических событий без
  `std::vector` и `std::deque`;
- `DifferenceForecastModel` - конечно-разностный прогноз первого или
  второго порядка;
- `CorrectionService` - вычисление ошибки и сравнение с порогами;
- `DecisionMaker` - формирование реакции `NO_PREDICTION`, `NORMAL`,
  `WARNING` или `CRITICAL`;
- `ForecastCorrectionProcessor` - главный обработчик событий;
- `ProcessingStatistics` - статистику прогона;
- `EventGenerator` - линейный поток, поток со скачком и поток с шумом;
- CSV-сериализацию событий и результатов.

Для каждого входного события обработчик выполняет следующий шаг:

1. Берёт прогноз, рассчитанный ранее для данного типа события.
2. Сравнивает прогноз с фактическим значением.
3. Вычисляет ошибку и абсолютную ошибку.
4. Выбирает реакцию по порогам `WARNING` и `CRITICAL`.
5. Добавляет факт в историю соответствующего типа события.
6. Строит прогноз на следующий шаг уже по обновлённой истории.

История и ожидаемые прогнозы хранятся отдельно для каждого типа события.
Поэтому в одном потоке можно смешивать, например, загрузку CPU и
температуру: прогноз одного показателя не влияет на другой.

События поступают из `LazySequence<Event>` через
`LazySequenceReadStream<Event>`, а результаты каждого шага записываются
через `SequenceWriteStream<ProcessingResult>`. По запросу пользователя
лог сохраняется в файл посредством `FileWriteStream<ProcessingResult>`.

### Демонстрация варианта

В главном меню пункт `11. Вариант 3.3: прогноз / коррекция событий`
предоставляет:

1. Настройку порядка модели, размера истории и порогов ошибок.
2. Ручной ввод событий.
3. Линейный поток с ожидаемо точным прогнозом.
4. Поток с резким скачком для демонстрации `CRITICAL`.
5. Поток с небольшим шумом.
6. Загрузку событий из CSV-файла.
7. Большой автоматический прогон со скачком и статистикой.

Любой сценарий можно выполнять пошагово. Для каждого события UI показывает
предыдущий прогноз, фактическое значение, ошибку, реакцию и следующий
прогноз. В автоматическом режиме выводятся количество реакций каждого типа,
средняя и максимальная абсолютные ошибки, время и число материализованных
элементов ленивой последовательности.

Формат CSV-файла входных событий:

```text
id,timestamp,type,value,source
1,1000,CPU_LOAD,40,server-1
2,2000,CPU_LOAD,45,server-1
```

```text
fourthlab/
|-- CMakeLists.txt
|-- main.cpp
|-- README.md
|-- classes/
|   |-- cpp/
|   |   `-- UI.cpp
|   `-- headers/
|       |-- Ordinal.h
```

Ключевые файлы:

- `classes/headers/Ordinal.h` - конечные и бесконечные длины;
- `classes/headers/ForecastCorrection.h` - реализация варианта 3.3;
- `classes/headers/LazySequence.h` - ленивая последовательность и её
  внутренние генераторы;
- `classes/headers/Streams.h` - потоки чтения и записи;
- `classes/cpp/UI.cpp` - консольный интерфейс;
- `tests/lab4_tests.cpp` - тесты GoogleTest.

- добавление, удаление, вставку и объединение;
- `Map`, `Where`, `Reduce`, `ReduceFirstN` и `Take`;
- чтение и запись потоков;
- ошибки состояний потоков и файловые ошибки;
- нагрузочную обработку миллиона элементов.
- историю, прогноз первого и второго порядка, коррекцию и реакции;
- обработчик смешанных типов событий, генераторы событий и CSV-формат
  варианта 3.3.
- автоматический прогон большого ленивого потока событий варианта 3.3.
